Utveckling av mallen

För att utveckla mallen behöver SASS vara installerat.

SCSS-struktur

Något förenklat är strukturen för scss-filerna:

  1. uu-system-style.scss är startpunkten som laddar:
    1. UU Bootstrap Theme
    2. Bootstrap
    3. Våra filer, som inkluderas enligt SMACSS-strukturen i ordningen:
      1. Base
      2. Layout
      3. Module
      4. State
      5. Theme

Komma igång

Efter att SASS installeras görs all utveckling i filerna under scss efter att SASS startats: sass --watch uu-system-style.scss ../css/uu-system-style-developement.css Eller genom filen watch.sh: sh watch.sh // Mac batch watch.sh // PC

Ändringar av filerna i mappen scss genererar därefter output till filen uu-system-style-developement.css som inkluderas på lämplig utgvecklingssida (t.ex. i denna fil).

SASS-strategi

Vi använder SASS då det ger oss vissa fördelar, t.ex. möjligheten att enkelt generera en enda CSS-fil av en mer uppdelad och bättre strukturerad uppsättning filer.

Vi får nog diskutera lite kring vilka delar av SASS vi ska använda.

Nästling får kan användas i max en nivå och bara med html-element, pseudoklasser (ex. :hover) och pseudoelement (ex. ::after).

Systemspecifik CSS

Filerna ovan ska bara innehålla CSS/SASS som är gemensam för alla våra system. CSS som är specifik för ett visst system läggs istället i en systemspecifik CSS i systemet som läses in efter uu-system-style.css, exempel: <link rel="stylesheet" type="text/css" href="uu-system-style-1.0.css" /> <link rel="stylesheet" type="text/css" href="glis-style.css" />

Alternativt, om systemet har bra SASS-stöd, kan den systemspecifika stylingen läggas i en tema-filen i SMACSS-strukturen som beskrivs ovan.

Oavsett vilken metod som används för den systemspecifik CSS:en bör vi sträva efter att minimera systemspecifik CSS och istället försöka hitta generella lösningar som kan användas av flera system i den gemensamma CSS:en. Detta är dock inte alltid möjligt, t.ex. när vi har liten eller ingen makt över html-strukturen, men där det är möjligt att skapa generella lösningar bör vi göra det och då också dokumentera funktionen i Designsystemet så den kan användas av flera.
